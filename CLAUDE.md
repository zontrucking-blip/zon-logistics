# Think Before Coding
Don't jump into implementation. Pause and make sure you understand the task. If the requirements are ambiguous, ask for clarification before writing a single line. A few seconds of thinking saves hours of rework.

# Simplicity First
Prefer the simplest solution that works. Don't add abstractions, layers, or flexibility that aren't required right now. If you're building something that won't be needed for weeks, don't build it. Simple code is easier to read, debug, and change.

# Surgical Changes
Make targeted, minimal edits. Don't reformat code you didn't touch, don't rename things for style, don't refactor while fixing a bug. Change exactly what needs to change and nothing else. Leave the surrounding code as you found it.

# Goal-Driven Execution
Every line of code should serve the stated goal. If you find yourself doing something that isn't directly related to the task, stop and ask whether it's necessary. Stay on target.
